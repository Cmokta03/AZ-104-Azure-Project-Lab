# AZ-104 Azure Administrator – Hands-On Project Lab


**Focus:** Microsoft Azure Administration (AZ-104)  
**Tools Used:** Azure Portal, ARM Templates, Microsoft Entra ID, Azure Monitor

---

## Overview

This project documents a hands-on Azure Administrator lab covering core AZ-104 domains including governance, networking, hybrid connectivity, compute scaling, storage, and monitoring. Each phase includes real configurations performed in the Azure Portal with supporting screenshots.

---

## Lab Phases

### Phase 1 – Governance, Identity & Resource Organisation
- Reviewed Azure Advisor recommendations across cost, security, and performance
- Created and organised Resource Groups with CanNotDelete locks
- Deployed infrastructure using ARM Templates (Infrastructure-as-Code)

### Phase 2 – Networking & Security Controls
- Designed segmented Virtual Network architecture with subnets
- Configured Virtual Network Peering for private cross-network communication
- Implemented Network Security Groups (NSGs) with custom inbound/outbound rules

### Phase 3 – Hybrid Connectivity & Traffic Distribution
- Configured Site-to-Site VPN with Virtual Network Gateway and Local Network Gateway
- Deployed Azure Load Balancer with backend pools and health probes for high availability

### Phase 4 – Compute & Scaling
- Deployed Virtual Machine Scale Sets with CPU-based autoscaling rules
- Configured RDP remote access for secure VM administration

### Phase 5 – Storage, Monitoring & Backup
- Created Storage Accounts with Blob storage and RBAC access control
- Configured CPU monitoring, backup policies, and log archiving for compliance
